Punjab Kings Store Functional Test Report

Date: 2026-03-11

Base URL: https://store.punjabkingsipl.in/

Execution: live Playwright browser run with authenticated flow using the provided mobile number and OTP.

Login, wishlist, sorting, filtering, cart add/remove, quantity update, logout, and basic form validation were tested live. The store is usable for core shopping only after a delivery pincode is entered on the PDP. Screenshots are attached for the functional issues and important state changes.

Functional Findings

Severity Area URL Issue Evidence
Medium PLP quick add /products The PLP ADD TO CART CTA does not directly add the product to the cart. It expands an inline quick-buy block with style/size details instead. That is not necessarily broken, but it is a different flow than the button label suggests and can confuse users expecting a direct add. functional-plp-quick-add.png
Low Cart item removal affordance /cart/bag The initial remove trigger inside the cart is an unlabeled icon/button. Remove is possible, but the affordance is weak until the confirmation sheet appears. functional-cart-with-item.png
Low Order tracking validation copy /order-tracking Blank submit returns a generic Invalid Order Id message instead of clearer required-field guidance. functional-order-tracking-invalid.png
